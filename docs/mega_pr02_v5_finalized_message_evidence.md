# MEGA-PR-02 V5 — hardened finalized message evidence

This checkpoint extends the merged MEGA-PR-02 raw-state economic proof with the
V5 transaction-correctness contract for `IMPL-61` through `IMPL-68`.

It is additive and live-disabled. It does not sign, submit, consume permits,
release capital or promote paper/live readiness.

## Contract added

`src/execution/finalized_message_evidence_mega_pr02.py` introduces
`HardenedFinalizedMessageEvidence` and `HardenedFinalizedMessageAuthority`.

The authority can return `READY` only when all of these invariants hold:

1. the compiled message was produced by `HardenedExactCompiler` and the exact
   simulation boundary consumed the same object;
2. every ALT table fingerprint is derived from on-chain raw bytes and is not
   deactivated at the accepted context slot;
3. the serialized wire size is below the protocol ceiling of 1232 bytes;
4. quote, market, oracle and ALT evidence have positive rooted slots, and
   `min_context_slot` cannot be zero;
5. validation, provisional simulation, compute finalization, final simulation,
   fee quote and blockhash check form one non-regressing causal timeline;
6. provider id and genesis hash do not drift between rooted inputs, timeline
   evidence and ALT evidence;
7. PR-128 compute unit limit, compute unit price, loaded account data limit and
   landing cost cap are finalized and bound to the final message fingerprint;
8. the final blockhash is rechecked after final simulation/fee quotation and has
   enough remaining block-height margin;
9. raw snapshots cover every semantically mandatory economic account: payer,
   writable accounts, token/native accounts, temporary ATA/WSOL accounts,
   protocol vaults, banks, margin accounts, oracles and cleanup recipients;
10. permit issuer and signer authorization consume the same finalized evidence
    hash.
