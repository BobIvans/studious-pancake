# MEGA-PR D2 — real soak and release artifacts

D2 is the first practical follow-up after the merged MEGA-PR D preflight.  It
does not pretend that a multi-day paper soak has already happened.  It adds an
active bundle producer that consumes real files generated by a protected
operator/CI release job and emits one deterministic, reviewable JSON bundle.

## Active command

```bash
python scripts/d2_release_soak_bundle.py --manifest evidence/d2-input.json --output evidence/d2-bundle.json
```

`--require-ready` exits non-zero while any mandatory artifact or soak condition is
missing.

## Required evidence

The bundle requires the actual candidate release to provide:

- wheel hash;
- locked/offline wheelhouse hash;
- container image digest;
- SBOM;
- dependency graph;
- provenance/attestation;
- source/wheel parity report;
- immutable PolicyBundle artifact;
- provider/program contract snapshot;
- soak summary, event export, resource metrics and restart-recovery evidence.

The soak portion requires a pinned wheel, pinned image and pinned policy bundle,
at least the reviewed duration, no synthetic/fixture contamination, no sender or
signer reachability, no signatures/submissions, no reservation leaks, no data
gaps and proven restart/cancellation/resource recovery.

## Safety

- Live remains disabled.
- No sender, signer, submission, Jito or RPC send path is added.
- The script only reads local evidence files and hashes them.
- Secret-looking metadata is rejected.
- Synthetic or incomplete soak evidence remains blocked.

## Relationship to PR-175/176

PR-175/176 are still open in parallel and should later provide signed executable
provenance and hermetic qualification.  D2 deliberately avoids depending on those
unmerged branches; it creates the D-side release/soak bundle format they can feed
after merge.
