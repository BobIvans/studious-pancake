# PR-169 independent launch certification gate

This document describes the first implementation slice for PR-169.

## Scope

PR-169 is an independent assurance and launch-certification layer. It does not
enable live trading, signing, deployment or automatic release promotion.

The new `src.release_gate.independent_assurance` module models the minimum
evidence contract for launch:

- maintained end-to-end threat model;
- security invariant evidence;
- property state-machine evidence;
- coverage-guided fuzzing evidence;
- mutation testing evidence;
- differential/protocol evidence;
- static/dataflow analysis evidence;
- external penetration-test evidence;
- launch risk register;
- independent signoffs over the exact release digest.

## Certification rules

A launch package is approved only when all of the following hold:

1. every required evidence kind is present;
2. every evidence artifact passed and contains raw signed provenance;
3. evidence producer and verifier identities differ;
4. all required signoff roles approve the exact release digest;
5. the independent reviewer did not author release changes;
6. every required security invariant is listed;
7. critical/high risks are closed, not merely self-accepted.

## Non-goals

- No live sender.
- No RPC, Jito, Helius, Jupiter or MarginFi network call.
- No private key loader.
- No deployment controller.
- No replacement for PR-166/167/168.
- No claim that an external penetration test already happened.

## Operator use

Future release packaging should create an
`IndependentLaunchCertificationPackage` from signed reports generated by the
security toolchain and external reviewer. The package must be evaluated before
any meaningful-funds launch gate can declare the release certifiable.
