# PR-024 — CI quality gates

This change makes known runtime/static defects visible instead of globally
suppressing them.

## Enforced gates

- Flake8 critical errors across `arb_bot.py`, `src`, `scripts`, and `tests`;
- Black on the incremental maintained-file manifest;
- Mypy on the canonical production surface with explicit, owner-tagged debt;
- Bandit hard failure for high-severity/high-confidence findings;
- `pip-audit` against the pinned runtime requirements;
- compileall, launcher lifecycle smoke, import smoke, and offline tests;
- Gitleaks remains the mandatory repository-history secret scan.

`src/legacy_arb_bot.py` is the only F821 exception. It is recorded as a
non-importable quarantine and an architecture test prevents maintained code
from importing it.

The opt-in MarginFi test now checks a real RPC response, slot, account owner,
and binary account bytes. It does not claim IDL/layout conformance; that work
belongs to PR-027 and PR-028.

## Local commands

Full CI-equivalent verification:

```bash
python -m pip install -r requirements-dev.txt
python scripts/verify_repo.py
```

Offline development, explicitly skipping only the network vulnerability query:

```bash
python scripts/verify_repo.py --skip-dependency-audit
```
