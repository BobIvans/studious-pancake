# PR-195 V3 durable kernel, webhook intake and capital authority gate

This slice starts the revised V3 roadmap **PR-195 — Durable runtime kernel,
webhook intake and atomic capital authority** without changing any live, signer,
sender, RPC or provider path.

The V3 audit moves a large group of hidden failure modes into PR-195.  The
critical boundary is no longer whether individual tables or helper classes exist.
PR-195 must prove that each externally visible success maps to one durable,
replayable business invariant.

## Scope in this slice

This PR adds `src/pr195_durable_kernel_v3.py`, an offline acceptance contract
for the PR-195 merge package.  The contract covers:

- durable-before-ACK webhook intake;
- immutable chain-event identity that does not use mutable payload hash as the
  exactly-once key;
- atomic wallet capital authority with serializable reservation and fencing;
- serialized lifecycle writes, optimistic row-count checking and monotonic lease
  renewal;
- owned outbox retry ceiling and DLQ;
- validate-then-atomic-rename restore with authenticated manifest requirement;
- event replay that reproduces materialized attempt, reservation and submission
  receipt state.

The module emits deterministic `pr195.durable-kernel-v3.v1` JSON evidence and
fails closed until every required proof is explicitly claimed.

## Deliberate non-goals

This is not a full PR-195 implementation and does not claim production paper
readiness.  It does not:

- enable live trading;
- load a private key;
- construct, sign, simulate or submit transactions;
- call RPC, Jito, Helius, Jupiter, MarginFi or Kamino;
- replace the existing lifecycle database;
- merge into or modify the parallel PR-195 follow-up branch.
