# PR-026 — typed configuration, secrets and chain registry

## Supported configuration contract

The installed runtime loads exactly one immutable `RuntimeConfig` with this
precedence:

1. packaged fail-closed defaults (`src/resources/runtime.default.yaml`);
2. optional YAML (`--config-file` or `FLASHLOAN_CONFIG_FILE`);
3. explicit supported environment variables;
4. explicit CLI overrides.

Unknown YAML fields, malformed pubkeys, cluster identity drift, decimal/float
money, embedded URL credentials and inline secrets fail closed. Legacy
activation flags set to true are rejected rather than translated.

```bash
flashloan-bot --config-file config/runtime.yaml config dump --json
flashloan-bot --config-file config/runtime.yaml config doctor --json
flashloan-bot --config-file config/runtime.yaml config doctor --online --check-secrets
```

`config dump` is always redacted. A secret must be referenced as `env:NAME`,
`file:/absolute/path`, or `keychain:service/account`; the configuration model
never stores the resolved value. Jito activation additionally requires a secret
reference to an operator-issued UUID credential. The code never generates a
fake Jito auth key.

## Chain registry

`config/chain_registry.json` and the packaged copy contain only stable platform
programs/mints plus a dynamic MarginFi slot. MarginFi has no address default:
its program, group, margin account and bank set must be supplied explicitly.
PR-027 remains responsible for pinning the deployment, IDL and RPC evidence.

Offline startup validates every configured pubkey, the configured cluster
identity and the program allowlist. When `validation.verify_rpc_at_startup` is
true, startup also runs read-only `getGenesisHash` and `getMultipleAccounts`
attestations and checks expected account owners before constructing the runtime.

## Compatibility and non-goals

- `RuntimeConfig.strategy_modes`, `opportunity_queue_size`, and
  `shutdown_drain_timeout_seconds` preserve the current PR-023 composition-root
  interface while keeping the source object immutable.
- PR-026 does not activate a strategy, sender, MarginFi integration or live mode.
- Packaging/container work remains owned by PR-025.
- External-contract pinning and drift detection remain owned by PR-027.
